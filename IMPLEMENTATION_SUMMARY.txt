OVERVIEW
================================================================================

The Private UPC feature adds security code word verification to high-risk 
operations in OpenClaw. When enabled, users must enter a secret code word 
before executing dangerous commands like bash scripts, system restarts, or 
configuration changes.

================================================================================
PROTECTED OPERATIONS (10 HIGH-RISK TOOLS)
================================================================================

When UPC is enabled, these operations require verification:

1. bash.exec              - Execute shell commands
2. spawner.spawn         - Spawn processes
3. fs.write              - Write to filesystem
4. sessions.spawn        - Create new sessions
5. gateway.restart       - Restart gateway
6. gateway.config-set    - Modify configuration
7. secrets.reload        - Reload secrets
8. cron.create           - Create cron jobs
9. plugins.reload        - Reload plugins
10. node.connect         - Connect to nodes

================================================================================
API METHODS REFERENCE
================================================================================

All methods are in the "security.upc" namespace.

security.upc.set
  Purpose: Set or update UPC credential
  Request: { upc: string, enabled: boolean }
  Response: { success: boolean, message: string }
  Auth: User scope

security.upc.verify
  Purpose: Verify code word for challenge
  Request: { challengeId: string, credential: string }
  Response: { verified: boolean, message: string }
  Auth: User scope

security.upc.requestChallenge
  Purpose: Request challenge for operation
  Request: { toolName: string, taskDescription: string }
  Response: { challengeId: string, requiresVerification: boolean }
  Auth: User scope

security.upc.getStatus
  Purpose: Get current UPC status
  Request: {}
  Response: { enabled: boolean, hasUPC: boolean, ... }
  Auth: Admin scope
